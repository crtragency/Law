# نظام إدارة مكتب المحاماة ⚖️

نظام متكامل وآمن لإدارة مكتب محاماة: القضايا والملفات، الموكّلون، المهام والتواصل
بين الموظفين، والتقويم ومواعيد الجلسات — مع نظام صلاحيات ودخول لكل موظف يُنشئه
المدير.

مبني بـ **Next.js 15** + **TypeScript** + **Prisma** + **PostgreSQL**، وجاهز للنشر
على **Vercel**. الواجهة عربية بالكامل (من اليمين لليسار).

---

## المزايا

- 🔐 **دخول لكل موظف** — الحسابات يُنشئها المدير فقط (لا يوجد تسجيل ذاتي).
- 👤 **صلاحيات حسب الدور (RBAC)** — مدير، محامي، مساعد قانوني، سكرتير، محاسب.
- 📁 **إدارة القضايا** — رقم القضية، النوع، الحالة، المحكمة، المحامي المسؤول.
- 👥 **إدارة الموكّلين** — بيانات كاملة وربطهم بالقضايا.
- 📎 **مستندات القضايا** — أرشفة روابط/مراجع المستندات لكل قضية.
- ✅ **المهام** — توزيعها على الموظفين، متابعة الحالة، والتعليق عليها.
- 📅 **التقويم** — جلسات المحاكم والاجتماعات والمواعيد النهائية.
- 📜 **سجل التدقيق** — تسجيل كل عملية حساسة (من فعل ماذا ومتى).

---

## الأمان 🛡️

صُمِّم النظام مع مراعاة الأمان في كل طبقة:

| الطبقة | الإجراء |
|--------|---------|
| كلمات المرور | تشفير بـ **bcrypt** (12 جولة) — لا تُخزَّن أبداً كنص |
| الجلسات | توكن عشوائي (32 بايت) يُخزَّن **مُجزّأً (hashed)** في قاعدة البيانات — قابل للإلغاء الفوري |
| الكوكيز | `httpOnly` + `Secure` + `SameSite=Lax` (حماية من XSS و CSRF) |
| تسجيل الدخول | **تحديد معدل المحاولات** (5 محاولات/15 دقيقة) + حماية من هجمات التوقيت |
| المدخلات | تحقق كامل بـ **Zod** على كل عملية |
| الصلاحيات | فحص RBAC في كل Server Action وكل صفحة |
| CSRF | فحص Origin إضافي فوق حماية Next.js للـ Server Actions |
| الترويسات | CSP، HSTS، X-Frame-Options، nosniff، إلخ |
| التدقيق | سجل كامل بكل العمليات مع IP والوقت |

عند **تعطيل حساب** أو **تغيير كلمة مرور** موظف، تُلغى كل جلساته فوراً.

## التشغيل محلياً

### 1) المتطلبات
- Node.js 20 أو أحدث
- قاعدة بيانات PostgreSQL (محلية، أو مجانية على [Neon](https://neon.tech))

### 2) تثبيت الحزم
```bash
npm install
```

### 3) إعداد متغيرات البيئة
انسخ ملف المثال واملأ القيم:
```bash
cp .env.example .env.local
```
عدّل `.env.local`:
```env
DATABASE_URL="postgresql://user:password@localhost:5432/law_office"
ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="كلمة-مرور-قوية"
ADMIN_NAME="مدير المكتب"
```

### 4) تجهيز قاعدة البيانات
```bash
npm run db:push      # إنشاء الجداول
npm run db:seed      # إنشاء حساب المدير الأول
```

### 5) التشغيل
```bash
npm run dev
```
افتح <http://localhost:3000> وسجّل الدخول ببيانات المدير.

## النشر على Vercel (مع Supabase)

1. ارفع المشروع إلى مستودع على GitHub.
2. أنشئ مشروعاً على [Supabase](https://supabase.com) واختر كلمة مرور قوية لقاعدة البيانات.
3. من Supabase: **Project Settings → Database → Connection string** وانسخ رابط
   **Session pooler** (منفذ 5432، يعمل على Vercel ويدعم كل العمليات). استبدل
   `[YOUR-PASSWORD]` بكلمة مرور قاعدة البيانات.
4. من [Vercel](https://vercel.com) اختر **New Project** واربطه بالمستودع، وتأكد أن
   **Framework Preset = Next.js**.
5. في إعدادات المشروع على Vercel → **Environment Variables** أضف:
   - `DATABASE_URL` ← رابط الاتصال من Supabase.
6. اعمل **Deploy / Redeploy**. أثناء البناء تُنشأ الجداول تلقائياً
   (`prisma db push`).
7. افتح رابط الموقع → ستُوجَّه إلى صفحة **`/setup`** لإنشاء حساب المدير من المتصفح
   (تعمل مرة واحدة فقط).

> **ملاحظة عن Supabase:** إن استخدمت "Transaction pooler" (منفذ 6543) بدل الـ
> Session pooler، أضف أيضاً متغيراً باسم `DIRECT_URL` بالاتصال المباشر (منفذ 5432)
> ليُستخدم في إنشاء الجداول.

> الأمر `build` يشغّل `prisma generate` وإنشاء الجداول تلقائياً.

### بديل: إنشاء حساب المدير بالأوامر
بدل صفحة `/setup`، يمكنك ضبط `ADMIN_EMAIL` و `ADMIN_PASSWORD` و `ADMIN_NAME` ثم:
```bash
npm run db:seed
```

## الأدوار والصلاحيات

| الدور | الصلاحيات |
|-------|-----------|
| **مدير المكتب** (ADMIN) | كل شيء + إدارة الموظفين + سجل التدقيق |
| **محامي** (LAWYER) | القضايا، الموكّلون، المستندات، المهام، المواعيد |
| **مساعد قانوني** (PARALEGAL) | القضايا، المستندات، المهام، المواعيد |
| **سكرتير** (SECRETARY) | عرض القضايا، إدارة الموكّلين والمواعيد |
| **محاسب** (ACCOUNTANT) | عرض القضايا والموكّلين (للأغراض المالية) |

عدّل الصلاحيات من `src/lib/rbac.ts`.

## بنية المشروع

```
prisma/schema.prisma        مخطط قاعدة البيانات
prisma/seed.ts              إنشاء حساب المدير الأول
src/lib/                    منطق الأمان (auth, session, rbac, rate-limit, audit...)
src/middleware.ts           حماية المسارات على مستوى الحافة
src/app/login/              صفحة تسجيل الدخول
src/app/(dashboard)/        كل شاشات النظام بعد الدخول
  ├── dashboard/            الرئيسية والإحصائيات
  ├── cases/                القضايا والمستندات والملاحظات
  ├── clients/              الموكّلون
  ├── tasks/                المهام والتعليقات
  ├── calendar/             التقويم والمواعيد
  └── admin/                إدارة الموظفين وسجل التدقيق
```

## تطويرات مقترحة لاحقاً

- **رفع الملفات فعلياً** بدل الروابط (عبر Vercel Blob أو S3).
- **تحديد معدل موزّع** عبر Redis (Upstash) عند النشر على عدّة نسخ.
- إشعارات بريد للمهام والجلسات القادمة.
- تقارير ولوحات مالية للمحاسب.

## ملاحظة أمنية مهمة

- لا ترفع ملف `.env` أو `.env.local` إلى GitHub أبداً (مستثنى في `.gitignore`).
- استخدم كلمات مرور قوية لحساب المدير وقاعدة البيانات.
- فعّل النسخ الاحتياطي الدوري لقاعدة البيانات.
